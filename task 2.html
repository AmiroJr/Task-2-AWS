Amazon Web Services (AWS) offers a wide range of cloud computing services and products that can be used to build, deploy, and manage
applications and infrastructure.

Some of the most popular AWS solutions include

  • Amazon Elastic Compute Cloud (EC2)
  • Amazon simple storage
  • Amazon Relational Database Service
  • Amazon Elastic Container Service
  • Amazon Lambda

1. Amazon Elastic Compute Cloud (EC2)

Amazon Elastic Compute Cloud (EC2) is a web service that provides resizable compute capacity in the cloud. It allows you to launch virtual servers (called instances) to run your applications. You can choose from a variety of instance types, each with different amounts of compute, memory, and storage resources, to meet the requirements of your applications.

Functioning

  1. You create an Amazon Elastic Compute Cloud (EC2) instance, which is a virtual machine that runs in the cloud. You can choose from a variety of instance types and configurations, and you can create as many instances as you need.
  2. You can use the Amazon EC2 API, the AWS Management Console, or the AWS CLI to launch and manage your instances. You can also use a variety of tools and libraries, such as AWS CloudFormation, AWS CodePipeline, and the AWS SDKs, to automate the process of launching and managing your instances.
  3. You can use your instances to run a variety of applications, including websites, mobile apps, data analysis, machine learning, and more. You can also use EC2 to host your own applications or to access applications provided by other AWS services, such as Amazon RDS, Amazon Redshift, and Amazon Elasticsearch Service.
  4. Amazon EC2 stores your instances and data in a highly scalable and durable manner. It stores your data across multiple facilities and devices, and it replicates your data within and across regions to provide high availability and durability.
  5. You only pay for the resources you use, and you can scale up or down as needed to meet changing workloads. You can also set up policies to control access to your instances and data, such as allowing or denying access based on the user's identity or the type of request.

Different instance types availale on EC2

  1. General purpose: These instances are well-suited for a variety of workloads, including web and application servers, development and test environments, and small databases.
  2. Compute optimized: These instances are designed for workloads that require a high amount of compute power.
  3. Memory optimized: These instances are optimized for workloads that require a high amount of memory.
  4. Storage optimized: These instances are designed for workloads that require a high amount of local storage.
  5. GPU instances: These instances are designed for workloads that require hardware acceleration using graphics processing units (GPUs), such as machine learning and video encoding.
  6. ARM instances: These instances are based on ARM processors and are well-suited for workloads that are optimized for low power and cost, such as containerized microservices and web servers.

2. Amazon Simple Storage Service (S3)

Amazon Simple Storage Service (Amazon S3) is a object storage service that offers industry-leading scalability, data availability, security, and performance. This makes it a popular choice for storing and retrieving any amount of data, at any time, from anywhere on the web.

Functioning

  1. You create an Amazon S3 bucket, which is a logical container for storing your data. You can create as many buckets as you need, and each bucket can store an unlimited amount of data.
  2. You upload objects (e.g., documents, images, videos) to your bucket. Each object is stored as a file, and you can organize your objects using a hierarchical file structure called a "prefix."
  3. You can access your objects using a variety of methods, including the Amazon S3 API, the AWS Management Console, and third-party tools. You can also set up policies to control access to your objects, such as allowing or denying access based on the user's identity or the type of request.
  4. Amazon S3 stores your objects in a highly scalable and durable manner. It stores your data across multiple facilities and devices, and it replicates your data within and across regions to provide high availability and durability.
  5. You can choose from a variety of storage classes, including Standard, Standard-Infrequent Access, and Amazon S3 One Zone-Infrequent Access, to help you optimize performance and cost for your specific use case. You only pay for the storage you use, and you can scale up or down as needed.

The main types of Amazon S3 storage classes

  1. Standard: Standard storage is designed for data that is accessed frequently. It provides high durability (99.999999999% per year) and high availability, and it is suitable for a wide range of use cases, including websites, mobile apps, data lakes, and more.
  3. Standard-Infrequent Access (Standard-IA): Standard-IA storage is designed for data that is accessed less frequently, but requires rapid access when needed. It provides the same durability and availability as Standard storage, but at a lower price point. Standard-IA is a good option for storing data that needs to be retained for a long period of time, but is not accessed frequently.
  4. Amazon S3 One Zone-Infrequent Access (S3 One Zone-IA): S3 One Zone-IA storage is similar to Standard-IA, but it stores data in a single availability zone instead of multiple availability zones. This makes it a lower-cost option for storing data that is not as critical and does not need the same level of availability and durability as Standard or Standard-IA.
  5. Amazon S3 Intelligent-Tiering: S3 Intelligent-Tiering is a new storage class that automatically moves data to the most cost-effective access tier based on usage patterns. It is designed to help you optimize costs without having to monitor and transfer data manually.
  6. Amazon S3 Glacier: S3 Glacier is a secure, durable, and extremely low-cost storage class for data archiving and long-term backup. It is designed for data that is not accessed frequently, and it provides three retrieval options (Expedited, Standard, and Bulk) to meet different recovery time objectives.

3. Amazon Relational Database Service(RDS)

Amazon Relational Database Service (Amazon RDS) is a managed database service that makes it easy to set up, operate, and scale a relational database in the cloud. It provides cost-efficient and resizable capacity while automating time-consuming database administration tasks, such as provisioning, backups, patching, and monitoring.

Functioning

  1. You choose a database engine, such as MySQL, MariaDB, PostgreSQL, Oracle, or Amazon Aurora. You can also select the hardware and network configuration that best meets the needs of your database.
  2. You create a database instance, which is a fully managed database environment that runs in the cloud. You can create as many database instances as you need, and you can scale up or down as needed to meet changing workloads.
  3. You connect to your database instance using standard database tools and protocols, such as SQL, ODBC, and JDBC. You can also use the Amazon RDS API or the AWS Management Console to manage your database instance.
  4. Amazon RDS automatically handles tasks such as provisioning, backups, patching, and monitoring, so you can focus on developing and deploying your application. You can also set up policies to control access to your database, such as allowing or denying access based on the user's identity or the type of request.
  5. Amazon RDS stores your data in a highly scalable and durable manner. It stores your data across multiple facilities and devices, and it replicates your data within and across regions to provide high availability and durability.

The main types of Amazon RDS are the following:

  1. MySQL: MySQL is a popular open-source database engine that is widely used for web, mobile, and embedded applications. Amazon RDS for MySQL provides a fully managed MySQL database service, with support for InnoDB storage engine and all the latest MySQL features.
  2. MariaDB: MariaDB is a fork of MySQL with a similar syntax and feature set. Amazon RDS for MariaDB provides a fully managed MariaDB database service, with support for InnoDB storage engine and all the latest MariaDB features.
  3. PostgreSQL: PostgreSQL is a powerful, open-source object-relational database system that is widely used for a variety of applications. Amazon RDS for PostgreSQL provides a fully managed PostgreSQL database service, with support for all the latest PostgreSQL features.
  4. Oracle: Oracle is a proprietary database engine developed and supported by Oracle Corporation. Amazon RDS for Oracle provides a fully managed Oracle database service, with support for all the latest Oracle features.
  5. Amazon Aurora: Amazon Aurora is a fully managed, MySQL- and PostgreSQL-compatible database engine developed by Amazon Web Services. It combines the performance and availability of high-end commercial databases with the simplicity and cost-effectiveness of open-source databases.

4. Amazon Elastic Container Service

Amazon Elastic Container Service (Amazon ECS) is a fully managed container orchestration service that makes it easy to run, scale, and monitor containerized applications in the cloud. It is designed to support the needs of both developers and operations teams, and it allows you to use Docker containers to build and deploy applications.

Functioning

  1. You create an Amazon ECS cluster, which is a logical grouping of EC2 instances or AWS Fargate tasks that you use to run your containerized applications. You can create as many clusters as you need, and you can scale up or down as needed to meet changing workloads.
  2. You create a task definition, which is a blueprint that describes the containers, CPU and memory requirements, networking, and other settings for your application. You can create as many task definitions as you need, and you can update them as needed to change the configuration of your containers.
  3. You create a service, which is a logical grouping of tasks that you use to run your application. You can specify the desired number of tasks and the placement strategies for your service, and Amazon ECS will automatically schedule and manage the placement of your tasks across your cluster.
  4. You can use the Amazon ECS API, the AWS Management Console, or the AWS CLI to manage your cluster, tasks, and services. You can also use Amazon CloudWatch to monitor the health and performance of your containers.
  5. Amazon ECS automatically handles tasks such as provisioning, scaling, and monitoring, so you can focus on developing and deploying your application. You can also set up policies to control access to your cluster and data, such as allowing or denying access based on the user's identity or the type of request.

The main types of Amazon ECS are :

  1. Amazon ECS on EC2: Amazon ECS on EC2 is the original version of Amazon ECS, and it allows you to run your containers on a cluster of Amazon Elastic Compute Cloud (EC2) instances.
  2. AWS Fargate: AWS Fargate is a serverless compute engine for containers that allows you to run your containers without the need to manage EC2 instances. With AWS Fargate, you can focus on writing and deploying your application, and Amazon ECS will handle the rest.
  3. Amazon ECS Anywhere: Amazon ECS Anywhere is a new service that allows you to run Amazon ECS on your own infrastructure, such as on-premises data centers or other cloud platforms.

What are AWS Cost and Usage Reports ?

The AWS Cost and Usage Reports (AWS CUR) contains the most comprehensive set of cost and usage data available. You can use Cost and Usage Reports to publish your AWS billing reports to an Amazon Simple Storage Service (Amazon S3) bucket that you own.

You can receive reports that break down your costs by the hour, day, or month, by product or product resource, or by tags that you define yourself. AWS updates the report in your bucket once a day in comma-separated value (CSV) format. You can view the reports using spreadsheet software such as Microsoft Excel or Apache OpenOffice Calc, or access them from an application using the Amazon S3 API. AWS Cost and Usage Reports tracks your AWS usage and provides estimated charges associated with your account. Each report contains line items for each unique combination of AWS products, usage type, and operation that you use in your AWS account. You can customize the AWS Cost and Usage Reports to aggregate the information either by the hour, day, or month.

Amazon EC2 usage reports

AWS provides a free reporting tool called AWS Cost Explorer that enables you to analyse the cost and usage of your EC2 instances and the usage of your Reserved Instances. You can view data up to the last 13 months, and forecast how much you are likely to spend for the next three months. You can use Cost Explorer to see patterns in how much you spend on AWS resources over time, identify areas that need further inquiry, and see trends that you can use to understand your costs. You also can specify time ranges for the data, and view time data by day or by month.


Setting up an Amazon S3 bucket for Cost and Usage Reports

To receive billing reports, you must have an Amazon S3 bucket in your AWS account to receive and store your reports. When setting up a Cost and Usage Report in the billing console you can select an existing Amazon S3 bucket you own or setup up a new one. In either case, you’ll be asked to review and confirm the application of the following default bucket policy. Editing this policy in the Amazon S3 console or changing the bucket owner after you’ve setup a Cost and Usage Report will prevent AWS from being able to deliver your reports. Storing the billing reports data in your Amazon S3 bucket is billed at standard Amazon S3 rates.




Working with Lambda functions metrics

When your AWS Lambda function finishes processing an event, Lambda sends metrics about the invocation to Amazon CloudWatch. There is no charge for these metrics. On the CloudWatch console, you can build graphs and dashboards with these metrics. You can set alarms to respond to changes in utilization, performance, or error rates. Lambda sends metric data to CloudWatch in 1-minute intervals. If you want more immediate insight into your Lambda function, you can create high-resolution custom metrics as described in the Lambda operator guide. Charges apply for custom metrics and CloudWatch Alarms.

How AWS cost and usage report works

With AWS Cost and Usage Reports (CUR), you can review, itemize, and organize the most comprehensive cost and usage data for your account.

Case Study on AWS Web Development services

E-Travel

One of the top 10 online travel agencies in Europe, e-Travel operates 11 websites, including www.mytrip.com, www.airtickets24.com, and www.trip.ru. Since mid-2014, more than two million passengers worldwide have purchased flights, travel insurance, and car rental services through the Athens-based start-up. Launched in 2007, the 180-employee company has a significant presence in south-eastern Europe and Russia and is expanding into markets in Southeast Asia.

The challenge

E-Travel has three core business goals: quick time to market, expansion into new countries and regions, and continuous delivery of useful features for customers. Focusing on these has been central to the firm’s success, so when its existing IT infrastructure—a combination of collocated and managed hosting services—hampered the company’s ambitions, e-Travel knew it had to look differently at its IT setup. “As we grew, we found that our environment was slowing us down,” says Nick Bitounis, director of IT at e-Travel. “We were concentrating on provisioning hardware, but what we really wanted to do was help deliver a great experience to our customers.” Bitounis says process automation was key in enabling the start up’s 35-plus in-house developers to experiment and test new ideas instead of focusing on day-to-day IT management tasks. With a 24/7 operation serving customers in more than 60 countries, e-Travel also needed to ensure high availability as well as elasticity to cope with significant spikes in demand. In 2012, things came to a head, with several hardware failures that ultimately threatened the customer experience. “Our most pressing goal was to eliminate downtime altogether,” says Bitounis. e-Travel looked to the cloud to help it achieve the service continuity that was so vital to its business.

WHy AWS ?

e-Travel uses AWS to run all its back-end systems for flight bookings and ticketing, as well as its 11 customer-facing websites. The services sit within an Amazon Virtual Private Cloud (Amazon VPC), and include Amazon elastic cloud (Amazon EC2), with Elastic Load balancing for distributing loads between instances. Amazon SNS (Amazon S3) provides fully redundant storage, and the managed Amazon relational database (Amazon RDS) reduces manual database administration for the team. e-Travel uses Amazon SNS and Amazon SQS to pass data between applications. The firm’s developers make extensive use of tools for Amazon Web Services, including the AWS SDK for ruby and AWS SDK for .NET to help integrate the company’s applications with its AWS environment. Amazon cloud search provides data indexing for the company’s proprietary customer relationship management (CRM) application, while is central to Amazon kinesis streaming log data. e-Travel hopes to expand its use of Amazon red shift for management reporting and is currently exploring how it can use AWS Lambda to realize the benefits of a serverless architecture. The company has the Business support plan from AWS Support, which Bitounis says has been helpful. “When the need arises, the response times, feedback, and resolution have been good,” he says.

Benefits

Eliminating system downtime was a key driver for moving to the cloud. “With AWS, we have 99.99 percent availability, which helps us deliver a consistent, reliable experience for our customers,” says Bitounis. The company’s sites all experience fluctuations in demand on an hourly, daily, and seasonal basis, and the elasticity of AWS gives e-Travel the ability to deal with these spikes in traffic. Bitounis says, “By using AWS, we can scale flexibly without having to spend time manually adding servers or removing bottlenecks. The infrastructure can easily respond to changes in demand.”

Reddit on AWS

Reddit has been operating on AWS since 2009. Originally, to support data that powered Reddit.com ad the company’s native mobile apps, Reddit used a self-managed PostgreSQL database on Amazon EC2, which provides secure, resizable compute capacity in the cloud and Amazon Simple Storage Service (Amazon S3), an object storage service that offers industry-leading scalability, data availability, security, and performance. Amazon EC2 and Amazon S3 gave Reddit’s engineers complete control of their computing resources as well as easy-to-use management features that enabled them to organize their data and configure finely tuned access controls to meet business, organizational, and compliance requirements.

Reddit uses Amazon Aurora, managed by Amazon RDS, which automates time-consuming administration tasks such as hardware provisioning and database setup, patching, and backups. Amazon Aurora granted Reddit engineers the to replicate its databases from Amazon EC2 without the need for transformation. Furthermore, they used Amazon Aurora to build a longer-term strategy for how they would deal with data and do more strategic thinking for lon-term projects.

Migrating from self-managed PostgreSQL on Amazon EC2 to Aurora was a key component of Reddit’s success. It has resulted in not only higher database reliability, faster backup restoration and point-in-time recovery, and fast automated failovers but also improved productivity for engineers.

More information on Amazon Aurora
